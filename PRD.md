# プロダクト要求仕様書 (PRD): Carrel

## 1. プロジェクト概要
### 1.1 ビジョン
「日常」の喧騒から「非日常」の物語へ。ユーザーが今、最も求めている物語と出会うための「キャレル（自分だけの読書席）」のような静寂な空間を提供し、心にフィットする究極の1冊を提案する。

### 1.2 コアバリュー
- **没入感の提供:** 選択や比較といった日常的な「タスク」を排除し、直感だけで物語と出会う。
- **究極のパーソナライズ:** 過去の履歴と現在の感情を高度に融合させた選書。
- **引き算の美学:** 最小限の情報提示により、ユーザーの想像力の余白を残す。

## 2. ターゲットユーザー
- 読書習慣があるが、次に何を読むべきか迷っている人。
- 忙しい日常の中で、一時的に物語の世界へ逃避（没入）したい人。
- 自分の感性や今の気分を言語化することに疲れを感じている人。

## 3. 主要機能 (Key Features)
### 3.1 パーソナライズ・エンジン
- **ユーザーベースラインの構築:** 過去の読書履歴、読後感想、ユーザー属性（年齢、職業、生活環境等）を解析し、感性のベースラインを定義する。
- **除外フィルター:** ユーザーが既に読んだことのある本を提案から確実に排除する。
- **書籍解析:** 世界中の書籍（ベストセラーを含む）を対象に、そのトーン、テーマ、読後の感情を多次元ベクトル化して管理する。

### 3.2 直感的気分入力インターフェース
- **非言語入力:** テキストボックスを排除。画面上に漂う色、波紋、抽象的なイメージ、あるいは情景の断片を直感的に選択することで「今の気分」を入力する。
- **音声入力（サブ）:** 抽象イメージで表現しきれない場合のみ、控えめに配置された音声入力導線を利用可能にする。

### 3.3 究極の1択提案ロジック
- **多層マッチング:** 「感性のベースライン」×「今の気分」を掛け合わせ、ベクトル検索を用いて最適な1冊を特定する。
- **セレンディピティの演出:** 有名なベストセラーから知る人ぞ知る名著まで、ユーザーが未読であればフラットに候補に含める。

## 4. ユーザー体験 (User Experience)
### 4.1 UIデザイン原則
- **トーン＆マナー:** 明るく淡いオフホワイト調。清潔感と静寂を感じさせるデザイン。
- **情報の段階的開示:** 最初は視覚情報（表紙）のみ。ユーザーの意思（タップ）に応じて言語情報（タイトル・概要）を提示する。

### 4.2 ユーザーフロー
1. **気分入力:** アプリ起動後、漂う抽象的なイメージから「今の心」に近いものを直感でタップ。
2. **顕現:** 背景から1冊の本の表紙（書影）がゆっくりと浮かび上がってくる。
3. **開扉:** 表紙をタップすると、本が開くアニメーションと共にタイトル、著者、概要が表示される。
4. **接続:** 詳細情報の下部に、Amazon等の購入・詳細ページへのリンクを表示する。

## 5. 技術的要件

### 5.1 システム要件
- **ベクトルデータベース:** 書籍のトーンや感情属性を検索するためのDB。
- **LLM活用:** ユーザーの過去データおよび感想テキストの多次元解析、および抽象イメージから感情ベクトルへの変換。
- **アニメーション:** 表紙のフェードイン、本が開くトランジションの滑らかな実装。

### 5.2 技術スタック
- **Frontend:** Flutter Web (Canvasベースの滑らかなアニメーション重視)
- **Backend:** FastAPI (Python)

- 読了後の感想を再びシステムにフィードバックし、精度を向上させるループの構築。
